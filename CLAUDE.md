## Project Overview

Personal blog and portfolio website built with **Quarto**, deployed to GitHub Pages from the `docs/` directory. The site includes blog posts, project showcases, and educational DSA content.

## Build Commands

```bash
# Render the full site (generates output in docs/)
quarto render

# Preview with live reload
quarto preview

# Render a single post/page
quarto render posts/my-post/index.qmd
```

There is no package.json, Makefile, or test suite. Quarto is the sole build tool.

## Architecture

- **Source files:** `.qmd` (Quarto Markdown) files with YAML frontmatter
- **Output directory:** `docs/` — committed to git, served by GitHub Pages
- **Configuration:** `_quarto.yml` is the main config (site structure, navbar, sidebar, themes)
- **Metadata inheritance:** `_metadata.yml` files in `posts/`, `projects/`, and `dsa/` apply shared settings (e.g., `freeze: true`) to all content in those directories

### Content Structure

| Directory | Purpose |
|-----------|---------|
| `posts/` | Blog posts (each is a folder with `index.qmd` + assets) |
| `projects/` | Project showcases (same structure) |
| `dsa/` | Educational DSA content with sidebar navigation |
| `docs/` | Generated output — do not hand-edit |

### Key Root Files

- `_quarto.yml` — Site config, navbar, sidebar, theme settings
- `index.qmd` — Home page with auto-generated post listing
- `about.qmd` — About page
- `projects.qmd` — Projects listing page
- `styles.css` — Custom CSS overrides
- `scripts.js` — Custom JS (quote slideshow in footer)
- `footer.html` — Custom footer template

## Creating New Content

A new blog post is a directory under `posts/` containing an `index.qmd` with this frontmatter:

```yaml
---
title: "Post Title"
author: "Vikrant Mehta"
date: "YYYY-MM-DD"
categories: ["Category"]
description: "Brief description"
---
```

Posts automatically appear on the home page listing (sorted by date descending). The same pattern applies to `projects/`.

## Publishing Workflow

1. Create/edit `.qmd` source files
2. Run `quarto render`
3. Commit both source files and generated `docs/` output
4. Push to GitHub — Pages serves from `docs/`

## Notes

- `freeze: true` is set in content metadata — Quarto reuses previously rendered output unless the source changes
- Themes: Flatly (light) / Darkly (dark) via Bootstrap
- `docs/search.json` and `docs/listings.json` are auto-generated by Quarto for search and listing functionality
